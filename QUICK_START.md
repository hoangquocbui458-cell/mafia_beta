# Быстрый Старт

Практические шаги для повседневной работы с проектом.

## 1) Запуск приложения

Проект работает как статический фронт без сборки.

Вариант A:
1. Открой index.html в браузере.

Вариант B (удобнее для кеша и обновлений):
1. Запусти локальный сервер из корня репозитория.
2. Открой страницу по адресу локального сервера.

## 2) Что править для типовых задач

- Изменить тексты и атмосферные фразы:
    - config.js, секция GAME_CONFIG.PHRASES.
- Изменить параметры роли:
    - config.js, секция GAME_CONFIG.ROLES.
- Изменить порядок ночных ходов:
    - config.js, массив GAME_CONFIG.NIGHT_ROLE_ORDER.
- Изменить поведение роли:
    - roles.js, объект RoleHandlers.<RoleName>.
- Изменить логику игры:
    - game-engine.js.
- Изменить отображение и экраны:
    - ui-manager.js, index.html, style.css.

## 3) Как добавить новую роль

1. Добавь роль в GAME_CONFIG.ROLES в config.js.
2. Добавь фразы в GAME_CONFIG.PHRASES.rolePrompts в config.js.
3. Добавь обработчик роли в roles.js в RoleHandlers.
4. Если роль должна ходить ночью, добавь ее в GAME_CONFIG.NIGHT_ROLE_ORDER.
5. При необходимости добавь стили в style.css.

Минимальная форма обработчика роли:

```javascript
NewRole: {
        canSelectTarget(targetIdx, gameState) {
                return !gameState.players[targetIdx].isEliminated;
        },
        onNightAction(targetIdx, gameState) {
                return {
                        type: 'NEW_ROLE_ACTION',
                        target: targetIdx,
                        message: `Новая роль выбрала: <b>${gameState.getPlayerName(targetIdx)}</b>`
                };
        },
        getRestrictions() {
                return {};
        }
}
```

## 4) Проверка перед пушем

1. Прогони автотесты движка:

```bash
node autotest.js
```

2. Вручную проверь ключевые сценарии в браузере:
- экран настройки ролей и карточку рекомендации;
- очистку конструктора и возврат рекомендованной раскладки;
- ночной цикл, голосование, первая и вторая ничья;
- экспорт лога в txt.

## 5) Частые проблемы

- Не применились стили или тексты:
    - сделай жесткую перезагрузку страницы.
- Кнопка Раздать роли неактивна:
    - в конструкторе сейчас 0 спецролей.
- Роль Любовница не увеличивается:
    - сначала добавь хотя бы одного Детектива.
- Ошибка в ночном ходе:
    - проверь ограничения в RoleHandlers и в renderGameScreen.

## 6) Мини-шпаргалка по архитектуре

- config.js: данные и правила конфигурации.
- roles.js: доменная логика конкретных ролей.
- game-engine.js: состояние партии и переходы фаз.
- ui-manager.js: рендер экранов и модалок.
- script.js: публичные callbacks для index.html.
