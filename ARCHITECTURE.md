## 🏗️ Выбранная Архитектура: Modular Monolith

```
┌─────────────────────────────────┐
│     Presentation Layer          │
│  (HTML UI + Event Listeners)    │
└──────────────┬──────────────────┘
               │
┌──────────────┴──────────────────┐
│  Application Layer (script.js)   │
│  - Callbacks & Integration      │
└──────────────┬──────────────────┘
               │
    ┌──────────┴──────────┐
    │                     │
    ↓                     ↓
┌───────────────┐   ┌──────────────┐
│ Game Engine   │   │   UI Manager │
│game-engine.js │   │ ui-manager.js│
│               │   │              │
│ Pure Logic    │   │ Pure Rendering
│ No UI, No I/O │   │ No Logic
└───────┬───────┘   └──────┬───────┘
        │                  │
        └──────────┬───────┘
                   │
        ┌──────────┴──────────┐
        │                     │
        ↓                     ↓
    ┌────────────────┐  ┌──────────────┐
    │ Roles Module   │  │ Config DB    │
    │   roles.js     │  │  config.js   │
    │ Role-Specific  │  │ All Settings │
    │    Logic       │  │ & Phrases    │
    └────────────────┘  └──────────────┘
```

## 📋 Почему Это Решение Хорошо?

### 1️⃣ **Separation of Concerns (SoC)**

Каждый файл отвечает за ОДНУ область:

| Файл | Отвечает за | Не знает о |
|------|-------------|-----------|
| `config.js` | Данные конфигурации | Логике, UI |
| `roles.js` | Логика каждой роли | Остальных ролях |
| `game-engine.js` | Игровая логика | UI, конкретных ролях |
| `ui-manager.js` | Отрисовка UI | Игровой логике |
| `script.js` | Интеграция компонентов | Внутренней логике |

### 2️⃣ **Extensibility (Расширяемость)**

Добавить новую роль = **3 шага**:
1. Добавить в `config.js`
2. Добавить логику в `roles.js`
3. Готово! ✨

**Старый спо​соб:** Искать все упоминания роли по всему коду

## 🔄 Типичный Поток Добавления Новой Роли

```
DEVELOPER
   │
   ├─→ Редактирует config.js (добавить роль, фразы)
   │      └─→ GAME_CONFIG.ROLES.NewRole = {...}
   │      └─→ GAME_CONFIG.PHRASES.rolePrompts.NewRole = [...]
   │
   ├─→ Редактирует roles.js (логика роли)
   │      └─→ RoleHandlers.NewRole = {
   │          canSelectTarget() {...},
   │          onNightAction() {...},
   │          getRestrictions() {...}
   │      }
   │
   └─→ Опционально: добавить CSS в style.css
          └─→ .tag-NewRole { ... }

РЕЗУЛЬТАТ: Новая роль ПОЛНОСТЬЮ интегрирована!
```

## 🔗 Связи Между Модулями

```javascript
// config.js ← не зависит ни от кого
ConfigUtils.getRandomPhrase(...)

// roles.js ← зависит только от config.js
RoleUtils.canSelectTarget(...) // использует ConfigUtils

// game-engine.js ← зависит от roles.js и config.js
gameEngine.submitNightAction() // использует RoleUtils

// ui-manager.js ← зависит только от game-engine.js
UIManager.renderGameScreen() // читает gameEngine.players и т.д.

// script.js ← склеивает всё вместе
// Использует: gameEngine, UIManager, ConfigUtils, RoleUtils
```

**Нет циклических зависимостей!** ✅

## 📚 Паттерны, Которые Использованы

### 1. **Module Pattern**
```javascript
const ConfigUtils = {
    getRandomPhrase() { ... },
    getRoleInfo() { ... }
};
```

### 2. **Singleton Pattern**
```javascript
let gameEngine = new GameEngine();  // One instance forever
```

### 3. **Handler Pattern**
```javascript
RoleHandlers = {
    Doctor: { canSelectTarget() {...}, onNightAction() {...} },
    Mafia: { canSelectTarget() {...}, onNightAction() {...} }
};
```

### 4. **Strategy Pattern**
Каждая роль может иметь разную логику в `.canSelectTarget()`

## 🚨 Потенциальные Проблемы & Решения

| Проблема | Решение |
|----------|---------|
| "Что, если нет ночных ролей?" | GameEngine.getCurrentNightRole() вернет null |
| "Что, если две роли имеют одинаковый класс CSS?" | Использовать уникальные имена (RoleName в CamelCase) |
| "Что, если забыть добавить роль в NIGHT_ROLE_ORDER?" | Роль все еще будет работать (в конфиге), но не активируется ночью |
| "Как откатить изменения?" | script-old.js - резервная копия старого кода |

## 🎓 Что Изменится В Будущем?

Архитектура готова к:

1. **Добавлению React/Vue** - UI-логика отделена (ui-manager.js)
2. **Переходу на TypeScript** - легко типизировать каждый модуль
3. **Добавлению Сетевой Игры** - game-engine.js полностью независимый
4. **Кешированию Логики** - можно добавить слой кеша
5. **Логированию/Аналитике** - добавить в игровой цикл
